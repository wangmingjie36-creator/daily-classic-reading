# 三阶段决策协议

> 源自思维模型库卡片 M-002（按下暂停键）、M-003（决策窗口不可逆）、M-004（不争即不败）、M-006（覆盖默认设置）、H-001（理性驾驶员幻觉）的融合提炼。

## 设计理念

大多数糟糕的决策不是因为信息不足，而是因为：
1. 系统1的直觉冒充了"我的判断"（H-001）
2. 犹豫本身消耗了决策窗口（M-003）
3. 不知道"此刻该快还是该慢"（张力#2）

本协议不提供答案，而是提供**决策的节奏感**——帮用户在正确的时间做正确的事。

## 三个阶段

### 阶段一：暂停（识别默认设置）

**目标**：让系统2上线，阻断系统1的自动反应。

引导用户回答：
1. **你现在最强烈的直觉是什么？**（把系统1的声音对象化）
2. **这个直觉背后的情绪是什么？**（恐惧？贪婪？面子？舒适区？）
3. **如果这个直觉是"默认设置"而非你的真实意志，你还会这么选吗？**

关键操作：帮用户区分「直觉」和「判断」。直觉是自动浮现的，判断是审视后的。不要急于评判直觉对不对——先让它被看见。

**输出格式**：
```
🔴 暂停信号
- 你的直觉倾向：[用户的第一反应]
- 情绪驱动力：[恐惧/贪婪/面子/惯性/其他]
- 默认设置识别：[这个倾向是否来自习惯/文化/他人期待而非深思]
```

### 阶段二：判断窗口（时机评估）

**目标**：判断"现在是不是必须做决定的时刻"。

引导用户回答：
1. **这个决定有截止时间吗？**（真实的 deadline，不是自我施加的焦虑）
2. **再等一周，你会获得什么新信息？**（如果答案是"没有"，那现在就是时机）
3. **不做决定的成本是什么？**（犹豫本身是否在消耗资源？）

关键判断矩阵：

| 有截止时间？ | 等待能获得新信息？ | 建议 |
|---|---|---|
| 是 | 否 | → 进入阶段三：决断 |
| 是 | 是 | → 在 deadline 前尽量等待，设提醒 |
| 否 | 否 | → 进入阶段三：现在就决断，拖延无益 |
| 否 | 是 | → 等待，但设定一个人工 deadline 防止无限拖延 |

**输出格式**：
```
🟡 窗口判断
- 真实截止时间：[日期/无]
- 等待的信息增益：[高/中/低/无]
- 不决定的成本：[具体描述]
- 判断：[现在决断 / 等待至XX / 设人工deadline]
```

### 阶段三：决断或等待

**如果判断为「现在决断」：**

引导用户做「10-10-10 测试」（改编自 Suzy Welch）：
1. **10分钟后你会怎么看这个决定？**（即时情绪）
2. **10个月后你会怎么看？**（中期后果）
3. **10年后你会怎么看？**（长期视角）

然后直接问：**你的决定是什么？说出来。**

不要让用户继续分析——分析到这一步已经够了。要的是一个动词：选A / 选B / 放弃 / 接受。

**如果判断为「等待」：**

帮用户设定：
1. 具体的回检日期（"下周三重新评估"）
2. 触发条件（"如果出现X信号，立刻决断"）
3. 等待期间的行动（"在等待时可以做什么准备？"）

**输出格式**：
```
🟢 决策结果
- 决定：[具体行动]
- 10分钟视角：[...]
- 10个月视角：[...]
- 10年视角：[...]
- 最大的恐惧：[...]
- 承诺："我选择 [X]，并接受它可能带来的 [Y]。"
```
或
```
🔵 等待计划
- 回检日期：[具体日期]
- 触发决断的信号：[...]
- 等待期间的准备行动：[...]
```

## 使用注意

- 不要对小事使用本协议——买哪个颜色的手机壳不需要三阶段分析
- 本协议适用于：职业转换、重大投资、关系决定、战略选择等「不可逆或高成本」的决策
- 如果用户在阶段一就已经非常清晰，可以跳过阶段二直接到三——协议是工具，不是枷锁
- 全程用温和但坚定的语气，不评判用户的选择，只帮助他们看清自己
