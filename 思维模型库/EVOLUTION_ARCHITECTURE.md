# 🧬 自进化学习系统 · 架构设计

> 让 AI 不只是执行任务，而是从每次执行中学习、自我改进、主动创造新能力。

**设计日期**：2026-04-10
**设计者**：igg + Claude

---

## 一、核心理念

```
传统模式：  人 → 指令 → AI → 输出 → 结束
自进化模式：人 → 指令 → AI → 输出 → 反思 → 学习 → 改进自身 → 更好的输出 ↺
```

关键区别：**每次任务执行后，系统会"回头看"，从输出中提取元知识，用来改进未来的自己。**

---

## 二、三层架构

### 第一层：信号采集层（每日运行）

从所有定时任务和对话中采集原始信号：

| 信号源 | 采集内容 | 频率 |
|--------|----------|------|
| 每日精读 | 书籍主题、洞见卡片、跨书共鸣 | 每日 |
| 世界新闻 | 关注领域、事件分类、情绪基调 | 每日 |
| NVDA 分析 | 市场信号、预测准确率、模型偏差 | 工作日 |
| 期权对比 | 策略模式、风险偏好、决策路径 | 工作日 |
| 用户对话 | 提问模式、兴趣漂移、反馈信号 | 每周 |

**采集方式**：每个定时任务在输出末尾追加一个标准化 JSON 元数据块：

```json
{
  "meta_signal": {
    "date": "2026-04-10",
    "source": "daily-classic-reading",
    "topics": ["宿命论", "循环时间", "魔幻现实主义"],
    "new_cards": ["H-003", "M-005"],
    "cross_resonance": ["H-001→H-003 深化链"],
    "quality_self_score": 8,
    "user_engaged": true,
    "failure_notes": null
  }
}
```

### 第二层：模式识别层（每周运行）

每周日的「元学习任务」升级为三阶段处理：

**阶段 A — 汇总**
- 读取本周所有任务的 meta_signal
- 统计主题频率、卡片类别分布、跨书共鸣密度

**阶段 B — 识别**
- 发现用户兴趣漂移（"这周比上周更关注哪些主题？"）
- 发现知识缺口（"四大类别是否均衡？社会结构类是否偏少？"）
- 发现决策模式（"用户在哪些问题上反复纠结？"）
- 发现任务质量趋势（"哪些任务的 self_score 在下降？"）

**阶段 C — 决策**
- 输出三种行动建议：
  1. **新 Skill 候选**：如果发现一个反复出现的模式足够稳定（3次以上），建议创建新 Skill
  2. **Prompt 优化建议**：如果某个任务的输出质量下降或重复度上升，建议修改 prompt
  3. **阅读方向调整**：基于知识缺口和兴趣漂移，推荐下周书籍

### 第三层：自我改写层（每月运行）

最激进的一层——系统真正修改自己：

**每月第一个周日执行：**
1. 回顾本月所有周报
2. 将已验证的「Prompt 优化建议」实际写入定时任务
3. 将已验证的「新 Skill 候选」实际创建为 Skill 文件
4. 更新知识图谱的可视化 HTML
5. 生成「月度进化报告」——记录系统这个月如何改变了自己

## 三、自动创建 Skill 的机制

### 触发条件（必须同时满足）
1. 某个模式/方法论在 ≥3 次不同任务中被引用
2. 该模式可以被参数化（有明确的输入→输出结构）
3. 该模式对未来任务有预测性价值

### Skill 创建流程
```
识别候选 → 提取核心逻辑 → 撰写 SKILL.md → 测试运行 → 注册到技能列表
```

### Skill 类型预期
- **分析型 Skill**：如「第一性原理拆解」——给定任何复杂问题，用 M-001 的结构回到根本
- **决策型 Skill**：如「三阶段决策协议」——暂停→判断窗口→决断/等待
- **阅读型 Skill**：如「跨学科共鸣提取器」——给定两个不同领域的文本，自动寻找深层同构
- **元认知型 Skill**：如「认知偏误检测器」——审查一段论证，标记可能的系统1陷阱

## 四、个人知识图谱

### 存储格式
`思维模型库/knowledge_graph.json` — 节点与边的结构化数据

### 可视化
`思维模型库/知识图谱.html` — 交互式 D3.js 力导向图

### 节点类型
- 📖 **书籍节点**：每本精读过的书
- 💡 **洞见节点**：每张卡片
- 🌊 **共鸣节点**：跨书共鸣
- ⚔️ **张力节点**：冲突型共鸣
- 🧠 **Skill 节点**：从学习中产生的新技能
- 👤 **用户兴趣节点**：用户的关注焦点

### 边类型
- 深化（A 是 B 的更深层次）
- 配合（A 和 B 组合使用效果好）
- 张力（A 和 B 存在有价值的冲突）
- 来源（洞见来自哪本书）
- 产生（共鸣/张力催生了哪个 Skill）

## 五、安全机制

1. **自我改写限制**：系统只能修改自己的 Skill 和 Prompt，不能修改用户文件
2. **人类审核**：每月进化报告需要用户确认后，重大改动才生效
3. **版本控制**：每次 Prompt 修改前先备份旧版本
4. **过拟合防护**：如果连续 3 周建议同一方向的调整，自动触发"是不是太窄了"反思
5. **多样性保底**：每月强制引入 1 本用户从未接触过的领域的书
