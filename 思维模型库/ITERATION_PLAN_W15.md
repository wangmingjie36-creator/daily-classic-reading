## 一、系统架构升级（5项改动）

### 1.1 🔴 紧急修复：路径硬编码问题

**问题**：`weekly-meta-learning` 任务的 prompt 中硬编码了旧 session 路径 `/sessions/dazzling-trusting-galileo/`，导致每次运行需要 fallback 到当前 session 路径。

**修复方案**：将所有路径改为**相对于挂载目录的动态发现**。新 prompt 先执行 `ls /sessions/*/mnt/每日阅读经典/` 来定位实际路径，而不是硬编码。

**状态**：✅ 已在本次更新中修复（见下方 weekly-meta-learning 新 prompt）

### 1.2 🔴 关键升级：每日精读的三级降级机制

**问题**：当前 `daily-classic-reading` 成功率仅 67%，content filtering 是唯一失败原因。

**方案**：引入「主选 → 备选 → 安全兜底」三级机制：

```
第一级（主选）：根据 learning_memory.json 的知识缺口推荐
第二级（备选）：如触发 filtering，自动切换到同类别的安全书目
第三级（兜底）：如连续3次失败，使用「安全书单」中的现代非虚构
```

**安全书单**（经验证不触发 filtering 的高质量书目）：

| 类别 | 安全书目 |
|------|---------|
| 哲学思想 | 《沉思录》《理想国》《论自由》《正义论》 |
| 科学认知 | 《时间简史》《自私的基因》《人类简史》《从一到无穷大》 |
| 历史社科 | 《枪炮病菌与钢铁》《国富论》《君主论》《文明的冲突》 |
| 文学名著 | 《百年孤独》《变形记》《局外人》《1984》《动物农场》 |

**预期效果**：成功率从 67% → 95%+

**状态**：✅ 已写入新 prompt（见下方）

### 1.3 🟡 重要升级：元学习任务的增量模式

**问题**：当前元学习任务每次全量重写 INDEX.md 和 learning_memory.json，随着数据增长会越来越慢且容易丢失历史数据。

**方案**：改为**增量追加**模式：
- 读取上周周报作为 baseline
- 只分析本周新增内容（通过文件修改日期过滤）
- INDEX.md 追加新共鸣而非重写
- learning_memory.json 用 append 而非 overwrite

**状态**：✅ 已在新 prompt 中实现

### 1.4 🟡 重要新增：任务间通信协议

**问题**：daily-classic-reading 的输出没有标准化元数据，元学习任务需要从 Markdown 正文中"猜"信息。

**方案**：在 daily-classic-reading 的 prompt 末尾要求生成标准信号块：

```json
<!-- META_SIGNAL
{
  "date": "2026-04-13",
  "book": "书名",
  "category": "四大类别之一",
  "topics": ["主题1", "主题2"],
  "cards_suggested": ["P-005", "H-005"],
  "quality_self_score": 8,
  "content_filtering_retries": 0
}
META_SIGNAL -->
```

元学习任务可以直接解析这个块，而不需要从正文推断。

**状态**：✅ 已写入新 daily-classic-reading prompt

### 1.5 🟢 优化：月度自进化任务的前置检查

**问题**：`monthly-self-evolution` 任务（每月1号 10:00）目前还没有运行过，需要确保它能正确读取所有周报并执行 Skill 创建。

## 二、阅读选书策略升级

### 2.1 知识缺口驱动的选书算法

当前系统的选书逻辑比较随机。升级为**知识缺口驱动**：

```
每日选书算法：
1. 读取 learning_memory.json 的 category_balance
2. 找到数量最少的类别
3. 在该类别的安全书单中选择未读过的书
4. 如果该类别已均衡，则优先选择 knowledge_gaps_detected 中提到的方向
5. 每周保证 4 个类别各至少出现 1 次
```

### 2.3 Content Filtering 规避策略

**高风险触发源**（已验证）：
- 中国古典诗词（前赤壁赋、唐诗宋词）
- 涉及暴力/色情描写的文学原文（战争场景的直接引用）
- 宗教经典的某些段落

**低风险策略**：
- 优先选择「分析性段落」而非「叙事性段落」
- 引用原文时用「节选+省略」而非全文
- 西方经典的英文原文+中文翻译模式（双语降低触发概率）
- 在 prompt 中明确指示：「如果内容被过滤，不要重试同一本书，立即切换到备选」

## 三、卡片网络深化策略

### 3.1 META 卡片生成规则（正式化）

| 条件 | 规则 |
|------|------|
| 触发阈值 | 某条共鸣涉及 ≥3 本书 |
| 命名规则 | META-XXX_概念名.md |
| 必含字段 | 多书证据表、深层同构分析、与现有卡片关系、非虚无主义声明 |
| 每周上限 | 最多创建 2 张（防止通胀） |
| 质量门槛 | 必须包含至少一个「调和路径」——不能只是指出相同，要说明为什么相同很重要 |

### 3.2 卡片连接密度提升方案

**当前问题**：M-005（宿命叙事反转）和 S-001（示弱进攻）是低连接度节点。

**提升策略**：

1. **每日精读时主动连接**：在 daily-classic-reading 的 prompt 中增加指令——每次生成新卡片时，必须指出它与现有 ≥2 张卡片的关系
2. **每周元学习时检查孤岛**：识别连接度 <2 的卡片，在下周阅读中优先寻找连接点
3. **引入「反向连接」机制**：当新卡片挑战旧卡片时，在旧卡片中追加「被挑战」标记

### 3.3 卡片生命周期管理

| 阶段 | 条件 | 动作 |
|------|------|------|
| 活跃 | 被引用 ≥2 次/月 | 保持 |
| 休眠 | 连续 2 月未引用 | 标记为休眠 |
| 存档 | 连续 3 月未引用 | 移至 archive/ |
| 升华 | 被 META 卡片吸收 | 保留但标记「已升华至 META-XXX」|

### 3.4 新增维度建议

当前四大维度（P/H/M/S）在第一周运转良好，但随着书目扩展，可能需要第五维度：

**建议新增：E（认识论/方法论元层）**— 关于「如何获得知识」本身的洞见

- 例：柏拉图的洞穴寓言 → E-001「感官经验的不可靠性」
- 例：卡尔·波普尔的证伪主义 → E-002「可证伪性是科学的边界」
- 暂不创建，等积累 3+ 个候选后再正式启用
